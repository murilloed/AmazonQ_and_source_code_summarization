RESUMO EM 60 SEGUNDOS

O Teste

Testamos por 7 dias uma ferramenta de monitoramento inteligente (Dynatrace) no sistema SARSB para verificar se conseguiríamos detectar problemas antes que os usuários reportassem.

O Resultado

Funcionou além do esperado. Durante a análise exploratória dos logs consolidados, identificamos que quase metade das requisições (48.7%) está falhando - situação que não era visível anteriormente.

A Recomendação

Considerando os resultados obtidos, sugere-se avaliar a contratação. O investimento se justifica pela capacidade de detecção proativa e prevenção de incidentes.

SITUAÇÃO ATUAL DO SARSB (Antes da PoC)

OPERANDO SEM VISIBILIDADE ADEQUADA

  • 48.7% das requisições HTTP resultando em erro
  • 2 funcionalidades com falhas não detectadas
  • 637 exceções técnicas registradas no período
  • Diagnóstico de problemas dependente de análise manual de logs
  • Equipe em modo reativo (respondendo a incidentes)

RISCO: Problemas impactando usuários sem detecção tempestiva

SITUAÇÃO FUTURA (Com ObaaS/Dynatrace)

VISIBILIDADE CONSOLIDADA EM TEMPO REAL

  • Problemas identificados rapidamente através de dashboards centralizados
  • Causa raiz mapeada automaticamente nas stack traces
  • Potencial para alertas proativos antes de impacto aos usuários
  • Equipe com foco em prevenção e melhoria contínua
  • Decisões técnicas baseadas em dados consolidados

RESULTADO: Sistema com maior previsibilidade e confiabilidade

AS 3 DESCOBERTAS MAIS CRÍTICAS

1. Alto Índice de Falhas em Requisições HTTP

O Problema:

  • 48.7% das requisições HTTP analisadas falharam (114 de 234 requisições no período)
  • Endpoints recebendo valores null e undefined como parâmetros
  • Usuários enfrentando telas de erro ou páginas em branco

Evidências Coletadas:

/prestador/null                              → 86 requisições (36.8%)
/listarSubCategoriaPrestador/null            → 24 requisições (10.3%)
/subcategoria/undefined                      → 3 requisições
/consultarPorUf/undefined                    → 1 requisição

Impacto no Serviço:

  • Municípios com dificuldade para reportar dados
  • Prestadores não completando processos de cadastro
  • Potencial impacto na qualidade dos dados coletados

Oportunidade de Melhoria:

  • Implementar validação de dados no frontend antes de chamadas à API
  • Adicionar tratamento de estados vazios/não inicializados
  • Estimativa de correção: 4-6 horas de desenvolvimento

2. Módulo de Contratos com Falha Total

O Problema:

  • Funcionalidade de gestão de atividades de contratos inoperante
  • 11 tentativas de acesso resultaram em erro (100% de falha)
  • Tabela sabtb_contrato_servicos_atividades não encontrada no schema do banco

Evidência Técnica:

Endpoint: /api/contrato/buscarAtividades
Erro: PSQLException - relation "sabtb_contrato_servicos_atividades" does not exist
Ocorrências: 11 tentativas no período
Status: Funcionalidade indisponível

Impacto no Serviço:

  • Impossibilidade de cadastrar ou consultar atividades contratuais
  • Interrupção de processos de negócio dependentes
  • Risco para conformidade e auditoria

Oportunidade de Melhoria:

  • Verificar e aplicar migrations pendentes do banco de dados
  • Validar integridade do schema em ambiente produtivo
  • Estimativa de correção: 1-2 horas (criação/restauração da tabela)

3. Volume Significativo de Exceções Não Monitoradas

O Problema:

  • 637 exceções Java registradas no período de 7 dias
  • Falta de visibilidade consolidada sobre erros em produção
  • Problemas técnicos acumulando sem tratamento

Distribuição das Exceções:

Validação de Tipos:
- MethodArgumentTypeMismatchException       → 228 ocorrências
- TypeMismatchException                     → 114 ocorrências
- ConversionFailedException                 → 114 ocorrências

Autenticação/Autorização:
- InsufficientAuthenticationException       → 67 ocorrências
- BadCredentialsException                   → 67 ocorrências
- InvalidTokenException                     → 63 ocorrências
- InvalidGrantException                     → 8 ocorrências
- AccessDeniedException                     → 6 ocorrências

Outras:
- IOException                               → 8 ocorrências

Impacto no Serviço:

  • Degradação progressiva da experiência do usuário
  • Dificuldade em priorizar correções sem visibilidade consolidada
  • Risco de incidentes mais graves não detectados

Solução:

  • Monitoramento contínuo com Dynatrace
  • Alertas configurados para exceções críticas
  • Análise de tendências para ação proativa

7. POR QUE ISSO IMPORTA PARA A ANA

Missão da ANA

A Agência Nacional de Águas tem a missão de:

  • Garantir segurança hídrica para população brasileira
  • Regular prestadores de serviço de saneamento
  • Monitorar recursos hídricos do país

SARSB é Peça-Chave

O Sistema SARSB apoia diretamente essa missão, sendo usado por:

  • Centenas de municípios para reporte regulatório
  • Prestadores de serviço para cumprimento de obrigações
  • Gestores da ANA para tomada de decisão baseada em dados

Falhas Têm Impacto no Serviço Público

Quando SARSB apresenta instabilidade:

  • Municípios enfrentam dificuldade para cumprir prazos regulatórios
  • Dados de saneamento podem ficar incompletos ou imprecisos
  • Decisões de gestão podem ser tomadas com informação parcial
  • Qualidade do serviço público à população é potencialmente comprometida

Observabilidade Como Requisito de Governança

Um sistema crítico para serviço público requer visibilidade adequada de sua operação e disponibilidade.

10. METODOLOGIA DA PoC

Para transparência, esclarecemos a metodologia utilizada:

Coleta de Dados:

  • Período: 7 dias (02 a 08/10/2025)
  • Instrumentação: Dynatrace OneAgent instalado no ambiente SARSB
  • Dados coletados: Logs de aplicação, métricas de requisições HTTP, traces de banco de dados, exceções Java

Análise:

  • Análise exploratória manual dos dados consolidados pelo Dynatrace
  • Identificação de padrões em 234 requisições HTTP registradas
  • Mapeamento de 637 exceções Java no período
  • Investigação de stack traces para identificação de causa raiz

Limitações:

  • Análise baseada em período curto (7 dias)
  • Não foram estabelecidos SLOs prévios para comparação quantitativa
  • Tempos de detecção "antes da PoC" são estimativas baseadas em experiência, não medições

Dados Brutos Disponíveis:

  • TopWebRequests.txt (234 requisições analisadas)
  • topoverview.ini (637 exceções catalogadas)
  • table-data.csv (5.480 linhas de log detalhado)
  • topDataBaseStatements.sql (165 queries únicas identificadas)
